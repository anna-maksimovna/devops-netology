# Домашнее задание к занятию "11.01 Введение в микросервисы"
### Задача 1: Интернет Магазин
Руководство крупного интернет магазина у которого постоянно растёт пользовательская база и количество заказов рассматривает возможность переделки своей внутренней ИТ системы на основе микросервисов.

Вас пригласили в качестве консультанта для оценки целесообразности перехода на микросервисную архитектуру.

Опишите какие выгоды может получить компания от перехода на микросервисную архитектуру и какие проблемы необходимо будет решить в первую очередь.



## Ответ
#### Выгоды от перехода на микросервисную архитектуру:
* Устойчивость к ошибкам ( уже как минимум не одна точка отказа, плюс есть различные техники по проработке ошибок (таймаут, прерыватель цепи и т.д.)
* Масштабируемость ( для высоконагруженных сервисов можно выделить несколько серверов, для менее нагруженных - меньше серверов)
* Простота развертывания ( можно чаще выкладывать обновления)
* Слабая связность ( изменения в одном сервисе по идее не должны влиять на другие сервисы)
* Простота замены ( для каждого сервиса можно выбирать свой стек технологий, причем если какая-то технология не подошла, то в данном случае легче перейти на другую)


#### Проблемы, которые необходимо будет решить в первую очередь:
* В первую очередь надо подумать на какие сервисы разделить монолит.
* Разделить команду по сервисам
* Определить как будут взаимодействовать микросервисы: синхронно/асинхронно, оркестрация/хореография, протоколы интеграции
* Выбрать способы межсервисной аутентификации и авторизации
* Настроить процессы CI/CD
* Настроить мониторинг системы, сбор логов
